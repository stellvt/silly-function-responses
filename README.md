# Silly Function Responses

License: this repository is available for review, auditing, discussion, and pull requests. Independent forks, separate builds, and derivative projects are prohibited. Read `LICENSE` and `CONTRIBUTING.md` before contributing.

Silly Function Responses adds assistant prefill support for Gemini 3.6 Flash in SillyTavern.

## Why this is needed

Gemini 3.6 Flash may return an empty response or a `400 Bad Request` error when a regular trailing assistant message is used as prefill.

The extension sends the same prefill as a synthetic function result instead.

## Features

- Custom assistant prefill.
- Native integration with **Start Reply With**.
- Optional capture of any plain trailing assistant message as the prefill.
- Support for **Show reply prefix in chat**.
- SillyTavern macro substitution, including `{{char}}` and `{{user}}`.
- Optional Gemini-only model filter.
- Optional OpenAI-compatible transport for native reverse proxies.
- Separate opt-in support for quiet requests, Continue, and Impersonate.

## How it works

The extension listens for SillyTavern's `CHAT_COMPLETION_SETTINGS_READY` event and modifies the final Chat Completion message list before it is sent.

It appends an OpenAI-compatible function exchange:

```text
assistant.tool_calls(name = silly_prefill)
tool(content = <assistant prefill>)
```

For direct native Google connections, SillyTavern converts this pair to Gemini `functionCall` and `functionResponse` parts.

Some reverse proxies accept the OpenAI tool-call representation but reject the equivalent parts on their native Gemini endpoint. For these proxies, enable **OpenAI transport for native reverse proxies**. The extension will route generated requests through the same proxy's OpenAI-compatible `/v1/chat/completions` endpoint without changing the selected SillyTavern connection. Leave the option disabled when the proxy accepts native Gemini function parts.

When this transport is enabled, unsupported `frequency_penalty` and `presence_penalty` fields are removed from the generated request.

When **Use Start Reply With** is enabled, the extension removes the matching trailing assistant prefill generated by SillyTavern and moves its content into the synthetic function result. **Show reply prefix in chat** continues to control whether SillyTavern displays that prefix in the rendered message.

When **Process any trailing assistant prefill** is enabled, consecutive non-empty plain `assistant` messages at the end are treated as the prefill. This supports prefills created by presets and other extensions, including prompts collapsed into a single assistant message. If **Use Start Reply With** is also enabled, its text is placed after the captured trailing prefill. An exact Start Reply With suffix is not duplicated. Custom prefill is used only when no trailing assistant prefill is captured and Start Reply With is disabled.

The extension registers its request handler last on `CHAT_COMPLETION_SETTINGS_READY`, after other extensions have loaded, so the synthetic function exchange is applied as the final frontend prompt transformation.

The synthetic assistant message uses an empty string for `content` rather than `null` for compatibility with stricter OpenAI-compatible request validators.

Prompt post-processing modes `merge`, `semi`, and `strict` are upgraded to their tool-preserving variants for the current request. The `Single user message` mode is not compatible with this extension.

## Installation

### SillyTavern extension installer

1. Open **Extensions**.
2. Select **Install extension**.
3. Enter the repository URL:

```text
https://github.com/stellvt/silly-function-responses
```

4. Complete the installation and reload SillyTavern.

### Git

```bash
cd SillyTavern/public/scripts/extensions/third-party
git clone https://github.com/stellvt/silly-function-responses.git
```

## Usage

### Custom prefill

1. Open **Extensions → Silly Function Responses**.
2. Enable the extension.
3. Enter the text in **Custom assistant prefill**.

### Start Reply With

1. Set **Start Reply With** in SillyTavern's **User Settings**.
2. Configure **Show reply prefix in chat** as desired.
3. Enable **Use Start Reply With** in the extension settings.

The custom prefill field is disabled while this mode is active.

### Trailing assistant prefills

Enable **Process any trailing assistant prefill** to convert final plain `assistant` messages produced by a preset or another extension. Consecutive messages are preserved in their original order, with the same double-newline boundary used by SillyTavern's Gemini conversion. Messages containing tool calls or multimodal content are left untouched.
