# Участие в разработке

Спасибо за желание улучшить Silly Function Responses.

Этот репозиторий открыт для просмотра, аудита, обсуждения и пулл реквестов. Это не проект с открытой лицензией в обычном смысле. Лицензия разрешает изменения только тогда, когда они предназначены для отправки обратно в этот репозиторий.

## Разрешённый порядок работы

Вы можете:

1. Создать форк репозитория на GitHub.
2. Создать ветку для изменения.
3. Изменить код для подготовки пулл реквеста.
4. Открыть пулл реквест в этот репозиторий.
5. Обсуждать и обновлять пулл реквест во время проверки.

Вы не можете:

1. Использовать форк как отдельный проект.
2. Публиковать релизы из форка.
3. Распространять изменённые копии.
4. Перепаковывать расширение под другим именем.
5. Копировать код в другой проект.

Форки GitHub разрешены как техническая часть подготовки пулл реквеста. Форк не даёт права поддерживать независимую версию расширения.

## Пулл реквесты

Делайте пулл реквесты сфокусированными.

Хороший пулл реквест обычно содержит:

1. Понятное описание изменения поведения.
2. Скриншоты или короткие заметки для UI-изменений.
3. Результаты ручной проверки в SillyTavern.
4. Небольшие коммиты с понятной целью.

Изменения формата function response, обработки префилла и совместимости с провайдерами должны объяснять пользовательский сценарий.

## Стиль кода

Следуйте текущему стилю расширения.

Полезные проверки:

```bash
npx eslint public/scripts/extensions/third-party/silly-function-responses/index.js
node --check public/scripts/extensions/third-party/silly-function-responses/index.js
node --test public/scripts/extensions/third-party/silly-function-responses/tests/lib.test.mjs
```

## Права на вклад

Отправляя пулл реквест, вы подтверждаете, что:

1. Вы написали вклад сами или имеете право его отправить.
2. Ваш вклад может быть включён в этот репозиторий на условиях лицензии репозитория.
3. Владелец репозитория может редактировать, публиковать, распространять и перелицензировать ваш вклад как часть Silly Function Responses.

## Задачи и предложения

Для сообщения об ошибке укажите:

1. Что вы ожидали увидеть.
2. Что произошло вместо этого.
3. Используемую модель и источник Chat Completion.
4. Значение режима префилла: собственный или Start Reply With.
5. Версию SillyTavern.
6. Ошибки из консоли браузера, если они есть.

Для предложения новой возможности сначала опишите пользовательский сценарий и ожидаемый формат исходящего запроса.
